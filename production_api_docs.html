API Overview

üöÄ Base URL

http://localhost:3000/api

üìã Response Format

All responses follow a consistent JSON format:

{ "success": boolean, "message": "string", "data": object | array | null, "error": "string (only on failure)" }

‚ö†Ô∏è Authentication

All endpoints require a userID field for tracking who performed the action. This is typically passed in the request body as createdByUserID, lastModifiedByUserID, or deletedByUserID.

Core Features

  • Project Management: Create and manage production projects with custom fields
  • Step Management: Define production steps with QC requirements and inventory needs
  • Batch Production: Organize units into production batches with tracking
  • Unit Tracking: Track individual units through each production step
  • Inventory Management: Manage raw materials and components with stock tracking
  • Reporting: Analytics and dashboards for production insights

üîÑ Error Handling

All endpoints return consistent error responses:

{ "success": false, "error": "Error message description", "details": "Detailed error information" }

Common HTTP status codes:

  • 400: Bad Request - Missing required fields or invalid data
  • 404: Not Found - Resource doesn't exist
  • 500: Internal Server Error - Database or server issues

üíæ Database Notes

  • Most deletions are "soft deletes" - records marked as inactive rather than removed
  • All timestamps are in UTC format
  • UUIDs are used for all primary keys
  • Decimal fields support up to 4 decimal places for inventory quantities
  • User tracking is required for all create/update/delete operations
